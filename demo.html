Cache Modes

Byte sits between your app and the provider. When a request comes in, Byte first checks its cache — if it can answer, it does so instantly at zero provider cost. The Mode controls how Byte decides whether two requests are "the same."

Exact fastest · safest
Byte-identical queries hit the cache. Zero false positives, zero embedding cost. Best when you want absolute accuracy and minimal latency.
Normalized fast · whitespace/case tolerant
Matches queries that differ only in whitespace, casing, or trivial punctuation. Still no embeddings, still fast.
Semantic paraphrase-aware
Uses embeddings so paraphrases and reworded questions can hit the same cache entry. Highest hit rate, but adds a small embedding cost on every miss and can occasionally return a loosely-similar answer.
Hybrid everything chained
Tries Exact → Normalized → Semantic in order. Maximum coverage with the safest matches taking priority.

Switching modes clears the cache so the new strategy starts clean.
